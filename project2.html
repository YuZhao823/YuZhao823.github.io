技术探针：针对垃圾分类认知程度的数据收集与分析

Design a technology probe to investigate how our fellow students do waste sorting in studio
设计了一个用来收集有关参与者对垃圾分类认知的数据物理装置，使用Arduino和相关传感器收集数据，并进行了数据可视化分析，作为用户行为分析模式的创新探讨。

The Context
背景介绍 1.0

技术探针（Technology Probes)是一种以技术为核心的探索方法，旨在：从工程的角度，它能于场域中直接做测试；从社会科学的角度,在真实的状况下去收集到科技的使用情况和其使用者的资讯；从设计的角度，它能启发使用者跟随设计师的思维，针对使用者的需求与想法，对新的科技去做发想。

The Context
背景介绍 1.1

垃圾分类对于刚到英国的留学生来说是一件较为困扰的问题，再加上语言和文化差异的影响，大家对如何正确回收垃圾可能存在不同的认知。为了响应“技术探针”的设计主题，设计了一个收集大家垃圾分类认识程度数据的交互装置。

Design Goals
设计目标

设计的主要目标为一下两点：

  • 是否大家对某一类垃圾存在的分类的认知错误
  • 是否大家在知道如何分类的情况下，仍然不愿意进行垃圾分类

Design Thinking
设计思考

日常生活垃圾多种多样，如果直接采取问卷调查或者实地观察统计，并不能直观地获取我们所需要的数据。使用“代币”的形式，可以减少参与者对垃圾的这一负面情绪的抵触，提高参与度的同时，也避免了使用图像识别的技术来对正式垃圾的分析。

Data Collection
数据收集 1.0

Arduino搭载重力传感器能够检测物体的重量，当不同的“垃圾代币”投入盒子后，通过记录重量数值的变化来自动获取所投入的代币所代表的垃圾类型。“垃圾代币”通过编码，可以相互组合，构造出“一次完整的垃圾制造过程”。当参与者拿到多个组合后，需要自行拆分，然后将其依次投入应当分类的盒子中，从而完成“垃圾分类”的模拟过程。

Data Collection
数据收集 1.1

通过Arduino记录每一次投入盒子所增加的重量，输出为所投入的垃圾类型。通过与盒子所代表的垃圾类型进行比对，即可判断参与者是否正确的进行垃圾分类。

Data Collection
数据收集 1.2

在硬件搭建和交互模式确定后，我们进行了参与模式的设计。参与模式的改变，能够探索出人们是否在知道如何分类的模式下，懒得去进行分类。第一天的参与模式是，将四类垃圾盒子放在一起，用户不需要进行位置的移动，主要探讨垃圾分类认知程度。第二天的参与模式是，将四类盒子分别放在大厅不同的位置，用户需要去找到这些盒子，探讨用户是否乐意多花费时间去进行垃圾分类工作。

Data Analysis
数据分析

通过对传感器数据的读取，初步获得了每个盒子的所拥有的“垃圾代币”类型，从而获取了大家对每一类垃圾的分类正确情况，获得了每类垃圾的正确率以及人们是否愿意进行垃圾分类的意愿分析结果。

Data Visiualization
数据可视化 1.0

通过对重量数据的分析，可以看出一个盒子在一段时间内所投入的垃圾的类型，直观的展现出来。

Data Visiualization
数据可视化 1.1

通过桑葚图可以看出每一类垃圾的走向和比例大小，直观看出哪一类的错误率。
